Here is Robo, maybe we will build him one day.

Robo has a central processing unit inside his head that reads computer programs and runs them - we call this a CPU. And he has some flash memory which is where all of those computer programs are stored.

This is what we call a terminal or command prompt. It is where we can enter commands on our computer and send them over to Robo. We type our command after the dollar sign and press enter to make it run. In this course we have buttons to help us do that.

Robo cannot understand the language we speak, he only understands Robo Language. Robo Language is software installed on Robo's flash memory, and the version Robo is currently using is Robo Language 1.0. It looks like this:



When we run the code from Robo Language 1.0 and watch how Robo moves we can see that each symbol is an instruction that Robo follows.

We can write a computer program to move Robo. This computer program only needs to contain the symbols (code) from Robo Language 1.0 in the order that we want Robo to move in. The software installed on Robo will read this code and Robo will follow our instructions.

Our program needs to be saved into a file ending with .bot. This tells the software on Robo that our file is a set of instructions meant for Robo.

Write a computer program called move.bot to get Robo from where he is standing to where we want him to go.

Usually programs are typed on computers with keyboards and usually programming languages are made up of those keyboard characters.

Update Robo's software to Robo Language 2.0 using the terminal.



$ update robo-lang


Updating robo-lang to version 2.0...
Update complete!


Robo Language 2.0 looks like this:

step 2
step -1
turn right
turn left


When we run the code from Robo Language 2.0 and watch how Robo moves we can see that each line of code is a complete instruction that Robo follows. step followed by a number moves Robo by that many spaces, which could be forward or back. turn followed by left or right spins Robo 90 degrees in a clockwise or anti-clockwise direction. 

step 2          // move forward two spaces
step -1         // move backward one space
turn right      // turn 90 degrees clockwise
turn left       // turn 90 degrees anti-clockwise

Two slashes (//) indicates a comment. Comments are not seen as code and do not get run as part of a computer program - they are used by programmers to write notes, often explaining what the code does.

Thinking Like a Computer

A computer is perfect. It will follow every instruction you give, and never complain, unlike your little brother or sister. This is what makes a computer so great, and also so annoying. You see it will follow every instruction you give, and never complain, not even if you tell it to blow itself up! So you better be careful what you tell it to do, because a computer is perfect.

We want to move Robo to the other side of the wall. It looks like we cannot move forward - we have to turn. If we tell Robo to step forward instead of turn he will definitely walk straight into the wall.

Ouch! Sorry Robo. We need to be careful with our instructions.

These instructions we are writing are part of Robo Language 1.0. Robo Language 1.0 is a complete set of instructions that Robo understands. We will learn more of the set as we go.

When we write a set of instructions for a computer it is called coding - we are writing code.

The instruction above tells Robo to step forward three spaces. We can use this type of notation with all of the Robo Language 1.0 instructions to shorten our code.

The instruction above tells Robo to jump over one space.

Perfect isn't he. Little Robo does what he's told.

So why is it that he follows instructions like that? Well at a very basic level Robo like any computer thinks in only two ways: on and off.

This is what we call binary. Often binary is explained as ones and zeroes, but all this is really telling us is that there are only two ways of doing something - either the thing is on (1) or it is off (0).

Have you ever seen a power switch with a 1 and 0 on it? This let's us know when the power is on or off and is the same way a computer works at a very basic level.

So how does this explain how a computer thinks because they can do a lot more than just turn on and off? That's true and it is because of a lot of offs and ons. Let's start with what one on/off looks like and build up.

A light switch is the best on/off example to look at and is literally on and off. When the switch is in the off position there is a space between two bits of metal and no electricity can flow in the circuit, and we have a dark room. When the switch is flicked on the two bits of metal touch, electricity begins to flow and we can see.

Now let's triple the amount of lights! One lightbulb is red, one is green and one is blue and they are all right next to each other and each light has its own switch. Light is cool because we can combine colours to make other colours, like mixing paint but sort of reverse. Anyway if we turn the red and green lights on and keep the blue light off we get yellow light. If we turn the red light off but keep the green and blue on we get cyan, a nice sky blue looking one.

Three switches, three ons and offs, multiple combinations - or codes?

Yes codes!

Red on, green on, blue off, is yellow. So in terms of switches: on, on, off = yellow. And remember sometimes on a switch, 1 is for on and 0 is for off. So 110 = yellow.

Here is the full colour model for visible light.

This is some of the ASCII alphabet used to represent letters in computers that use text. Each letter is coded in eight bits. A computer can decode the binary and match it to the correct letter.

0100 0010

0100 0101
0100 0110

The oppostite of decoding, figuring out the code, is encoding, creating the code.

This seems like a lot of work just to make some colours and letters. Luckily we don't have to go so deep inside a computer that we have to manually switch each little thing on and off - we have programming languages like Robo Language 1.0 that somebody has already figured out and given us some easy codes that do a million and one things underneath. But we know that we can go really deep and do it ourselves if we need to.

Good, now we know a computer is perfect and thinks nothing like us. Let's move on.

Looking Inside a Computer

A computer has stuff inside it and that stuff is called hardware. Is it hardware because it is hard, maybe. And it doesn't matter what computer you look inside because they all have something in common.

Remember Robo? In the beginning his head had just a CPU and some flash memory inside of it - having something that processes information and something that stores information is pretty common for a computer.

What computer should we look in? Your laptop? Your phone? Yes your phone might be a computer. Some people call them smartphones but if you can tell it to destroy itself and it will, it doesn't sound very smart. No, let us take a look inside a classic desktop computer - the parts are big and easy to get at.

Do not worry about remembering everything that follows - we will take our time to figure it out. But first we must name each part and build on that.

The central processing unit (CPU) reads and writes data from the hard drive and RAM, and controls everything in the computer.

The hard drive is where we store programs that we want to keep after the computer is turned off. It is a type of slow memory.

The RAM (random access memory) is a type of fast memory used to do things while the computer is turned on. When the computer is turned off everything on the RAM is gone.

The power supply converts power from the wall socket into something all of the other computer parts use, and feeds the power to all of the other parts.

The disc drives are used to read and write to optical media which are just CDs and DVDs. Discs use a fascinating form of binary that we will look at.

There are many more pieces to a desktop computer: ports for devices, sound and video stuff, network stuff. But the pieces above are the main ones.

To tie this chapter on hardware to the binary we learnt about earlier we are going to talk about CDs and DVDs.

A CD or DVD is a disc. They encode binary data in the form of physical pits and lands.

A disc drive reads the binary code by shooting a laser at the disc as it spins. The laser will bounce directly back if it hits a land - this means the binary data is reading 1 (or on). If the laser hits a pit the laser will bounce away and this means 0 (or off).

The code is sent to the CPU and the CPU follows those instructions like play the music or movie. It is a very fast and fascinating process - but most things inside computers are very fast and fascinating.

Using a Computer

A computer has stuff stored in its memory and that stuff is called software. Is it software because it is soft, no, its just not hardware. And its just a bunch of code, really that's it.

Remember Robo? In the beginning, he had computer programs stored in his flash memory - that was software and that was just code.

So where should we start? Well at the start. At the point where you press the on button. We call this booting up.

If we watch the screen when we press the power button we will see some interesting stuff. We might see some budget looking text and maybe an image. This will go away and a little while later a familiar logo will come up and then we will end up at our fancy login screen. We have experienced an important process.

What happened was everything - before then there was nothing.

That first budget looking part is the BIOS. It is a kind of software called firmware that checks hardware and gets it ready to go. It is the first piece of software to run and usually is stored on flash memory. Didn't Robo have some free flash memory?

We are looking at the BIOS but nothing else happened. Where is our fancy stuff? What we were expecting to show up next is called an operating system (OS).

An OS is all of that software we know and love. Windows, OS X and Linux are all popular operating systems. The second job of the BIOS is to run the OS which is usually stored on a hard drive. We need to install an OS on Robo's hard drive.
